# US-29 — DRF + JWT + drf-spectacular setup — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Install and configure the REST stack (DRF + simplejwt + django-filter + drf-spectacular) and expose `/api/v1/schema|docs|redoc/`, with reusable API test fixtures — no domain endpoints.

**Architecture:** Settings live in `settings/base.py` (3 DRF config dicts + env vars). A plain `settings/api_urls.py` module aggregates the API URL space under `/api/v1/`, mounting only the drf-spectacular views and carrying a placeholder for US-30+ per-app router includes. API test fixtures go in the existing `tests/conftest.py`.

**Tech Stack:** Django 6.0, Django REST Framework, `djangorestframework-simplejwt`, `django-filter`, `drf-spectacular`, `djangorestframework-stubs`; pytest / pytest-django.

    """Factory: ``auth_client(user)`` -> APIClient carrying a Bearer JWT for that user.

    The canonical authed-client pattern reused by all US-30+ API tests.
    """

    def _make(user):
        client = APIClient()
        access = RefreshToken.for_user(user).access_token

    return _make


@pytest.fixture(autouse=True)

    """Reset DRF throttle counters between tests (shared per-process LocMemCache)."""
    cache.clear()
    yield
    cache.clear()
