## 1. Decyzje brainstorming

| # | Pytanie | Decyzja |
|---|---|---|
| Q1 | **Gdzie żyje Pillow + MEDIA_URL/MEDIA_ROOT config?** | Bundle do US-10. `ImageField` wymaga Pillow w runtime (Django system check fail bez niego), więc dzielenie na US-10a/US-10b wprowadziłoby intermediate green state niemożliwy do osiągnięcia. Jedna PR, jeden cohesive change. |
| Q2 | **`Screening.booked_seats_count()` i pokrewne metody — co robić bez modelu Booking?** | Stubujemy: `booked_seats_count() → 0`, `available_seats_count() → hall.capacity - 0`, `is_in_past()` używa `timezone.now()`, `is_available() → not is_in_past() and available_seats_count() > 0`. US-18 podmieni stub `booked_seats_count` na real aggregation z CONFIRMED bookings — sygnatura metody się nie zmienia, więc M2 views (US-11/13/14) mogą jej używać bez warunkowych. |
| Q3 | **`Movie.get_absolute_url()` — implementować teraz?** | Defer do US-13. URL pattern `cinema:movie_detail` powstaje dopiero w US-13; teraz `reverse_lazy(...)` rzucałby `NoReverseMatch` przy każdym wywołaniu. Admin nie używa `get_absolute_url()` automatycznie (tylko jeśli adminie zarejestruje admin.View Site Link), więc brak metody w US-10 nikomu nie przeszkadza. |
| Q4 | **Factory_boy fabryki — gdzie i kiedy?** | `tests/cinema/factories.py` w US-10. Mirror pattern z `tests/accounts/factories.py`. Fabryki używane od razu w `test_models.py` US-10 oraz dostępne dla wszystkich późniejszych US M2 (US-11..US-14 view tests, US-15 admin tests, US-16 seed extension testing). |
| Q5 | **i18n verbose_names na model fields** | Każde pole dostaje `verbose_name = _("...")` z `gettext_lazy as _`. Spec §3 explicit wymaga tego ("Wszystkie pola modelowe mają zdefiniowane `verbose_name = _(...)` z `gettext_lazy`"). Mały upfront cost, brak refactor punktowego w FR-15 (i18n wiring). |

## 2. Architektura i lokalizacja

### 2.1 Model graph

```
Genre   ──┐
Actor   ──┼──> Movie ──> Screening ──> Hall
Director──┘    (M2M)     (FK CASCADE)   (FK PROTECT)
                          (FK CASCADE)
```

- **M2M:** `Movie.genres`, `Movie.actors`, `Movie.directors` → odpowiednio `Genre`/`Actor`/`Director`. `related_name="movies"` na każdej stronie.
- **FK CASCADE:** `Screening.movie` — usunięcie filmu kasuje wszystkie jego seanse. Sensowne dla wycofania filmu z kina.
- **FK PROTECT:** `Screening.hall` — usunięcie sali z planowanymi seansami rzuca `ProtectedError`. Chroni przed accidental hall removal.

## 3. Specyfikacja modeli

> Wszystkie modele importują `from decimal import Decimal`, `from django.core.validators import MinValueValidator`, `from django.db import models`, `from django.utils import timezone`, `from django.utils.translation import gettext_lazy as _`.

### 3.1 `Genre`

| Pole | Typ | Ograniczenia | verbose_name |
|---|---|---|---|
| `name` | CharField(50) | unique, not null | `_("name")` |

```python
class Meta:
    verbose_name = _("genre")
    verbose_name_plural = _("genres")
    ordering = ["name"]

def __str__(self) -> str:
    return self.name
```

class Meta:

Identyczna struktura jak `Actor` — `full_name` + `photo` (`upload_to="directors/"`) + `biography`. Osobny model (nie wspólny base) dla:
- jasnej semantyki w schemacie,

```python
class Meta:

    verbose_name_plural = _("directors")
    ordering = ["full_name"]

def __str__(self) -> str:
    return self.full_name
```

### 3.5 `Movie`

| Pole | Typ | Ograniczenia | verbose_name |
|---|---|---|---|
| `title` | CharField(200) | not null | `_("title")` |
| `description` | TextField | not null | `_("description")` |
| `release_date` | DateField | not null | `_("release date")` |
| `duration_minutes` | PositiveIntegerField | `MinValueValidator(1)` | `_("duration (minutes)")` |

```python

    verbose_name = _("movie")
    verbose_name_plural = _("movies")
    ordering = ["-release_date", "title"]

def __str__(self) -> str:
    return self.title

# get_absolute_url() — deferred to US-13 (cinema:movie_detail URL pattern doesn't exist yet)
```

### 3.6 `Screening`

| Pole | Typ | Ograniczenia | verbose_name |
|---|---|---|---|
| `movie` | ForeignKey → Movie | `on_delete=CASCADE`, `related_name="screenings"` | `_("movie")` |
| `hall` | ForeignKey → Hall | `on_delete=PROTECT` | `_("hall")` |
| `start_time` | DateTimeField | not null | `_("start time")` |
| `price` | DecimalField(6, 2) | `MinValueValidator(Decimal("0.01"))` | `_("price")` |

```python
class Meta:
    verbose_name = _("screening")
    verbose_name_plural = _("screenings")
    ordering = ["start_time"]

def __str__(self) -> str:
    return f"{self.movie.title} @ {self.start_time:%Y-%m-%d %H:%M}"

def booked_seats_count(self) -> int:
    # US-18 will sum seats_count from CONFIRMED bookings; until Booking exists no bookings exist.
    return 0

def available_seats_count(self) -> int:
    return self.hall.capacity - self.booked_seats_count()

def is_in_past(self) -> bool:
    return self.start_time <= timezone.now()

def is_available(self) -> bool:
    return not self.is_in_past() and self.available_seats_count() > 0
```

**Brak `clean()` w US-10.** Spec FR-3.7 nie wymaga walidacji na poziomie Screening (start_time może być w przeszłości — przydatne do backfilling). Walidacja "nie rezerwuj w przeszłości" siedzi w `Booking.clean()` w M3.

## 4. Konfiguracja Django

### 4.1 `pyproject.toml` — Pillow dependency

```toml
[project]
dependencies = [
    # ... existing entries ...
    "pillow>=11.0,<12",
]
```

User uruchamia `poetry add 'pillow>=11.0,<12'` — Poetry pisze constraint i regeneruje `poetry.lock`. Pillow 11.x ma wheels dla Python 3.13 (Windows + Linux dla CI).

**Production dep, nie dev:** `ImageField` walidacja uploadów dzieje się w runtime w każdym środowisku (admin upload, view form save), nie tylko w testach.

### 4.2 `settings/base.py` — MEDIA settings

Dodać pod sekcją `STATIC_URL` (~linia 88, zastąpić placeholder comment):

```python
# ─── Media (user uploads — Actor/Director photos, Movie posters) ────────────
MEDIA_URL = "media/"
MEDIA_ROOT = BASE_DIR / "media"
```

### 4.3 `settings/urls.py` — dev media serving

```python
from django.conf import settings
from django.conf.urls.static import static
from django.contrib import admin
from django.urls import include, path

urlpatterns = [
    path("admin/", admin.site.urls),
    path("accounts/", include("apps.accounts.urls", namespace="accounts")),
    path("", include("apps.cinema.urls", namespace="cinema")),
]

if settings.DEBUG:
    urlpatterns += static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)
```

`static()` zwraca `[]` gdy `DEBUG=False`, więc safe w module-level. Production serwuje `/media/` przez nginx/CDN — poza scope M2.

### 4.4 `.gitignore` + `media/.gitkeep`

Dodać do `.gitignore`:
```
media/*
!media/.gitkeep
```

Stworzyć pusty plik `media/.gitkeep` — zapewnia że `MEDIA_ROOT` istnieje po `git clone` (inaczej Django 500-uje przy próbie zapisu uploadów). Uploaded pliki są ignorowane przez git.

## 5. Strategia testów

### 5.1 `tests/cinema/factories.py`

Mirror pattern z `tests/accounts/factories.py`. 6 fabryk:

| Fabryka | Domyślne pola |
|---|---|
| `GenreFactory` | `name = Sequence(f"Genre {n}")`, `django_get_or_create=("name",)` |
| `ActorFactory` | `full_name = Faker("name", locale="pl_PL")`; photo blank, biography blank |
| `DirectorFactory` | `full_name = Faker("name", locale="pl_PL")`; photo blank, biography blank |
| `HallFactory` | `name = Sequence(f"Sala {n}")`, `django_get_or_create=("name",)`, `capacity=100` |
| `MovieFactory` | `title = Sequence(f"Film {n}")`, `description = Faker("paragraph")`, `release_date = Faker("date_this_year")`, `duration_minutes=120`, poster blank, trailer_url blank. M2M via `@post_generation` hooks dla genres/actors/directors (caller passes iterables). |
| `ScreeningFactory` | `movie = SubFactory(MovieFactory)`, `hall = SubFactory(HallFactory)`, `start_time = Faker("future_datetime", end_date="+30d", tzinfo=zoneinfo.ZoneInfo("Europe/Warsaw"))`, `price = Decimal("25.00")` |

**Note on ImageField factories:** photo/poster zostają puste domyślnie. Jeśli przyszły test będzie wymagać prawdziwego pliku, użyje `SimpleUploadedFile` lub `factory.django.ImageField()` — ale w US-10 wszystkie testy używają pustych ImageFields (testujemy że `blank=True` działa).

### 5.4 Co NIE testujemy

- Pillow ImageField upload roundtrip — out of scope dla model tests; smoke w admin (US-15) i w view tests (US-13)

### 5.5 Test ordering i fixtures

- Wszystkie testy niezależne — każdy z `@pytest.mark.django_db`
- Brak shared state między testami
- Fabryki dostarczają wszystkie dane testowe — żadnych hardcoded create calls poza testowanymi przypadkami brzegowymi (validators, uniqueness)

- (A) Dodajemy wszystkie 6 modeli w jednym commicie + jedną migrację, albo
- (B) Każdy model osobny commit, migracja `0001` regenerowana per commit (squash przed PR), albo
- (C) Migracja `0001_initial.py` w osobnym commit na końcu

Preferencja: **(A) lub (C)** — czysta migration history.
